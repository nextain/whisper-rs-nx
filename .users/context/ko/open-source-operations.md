# 오픈소스 운영 — whisper-rs-nx

whisper-rs-nx가 AI 네이티브 오픈소스 Rust 크레이트로서 어떻게 운영되는지.

**영문 미러**: `.users/context/open-source-operations.md`
**AI 컨텍스트 (SoT)**: `.agents/context/open-source-operations.yaml`

## 핵심 입장

**"AI에 대항하여 방어하지 말고, AI와 함께 설계하라."**

2025-2026년 대부분의 오픈소스 프로젝트가 AI 기여에 대항하여 방어하고 있습니다. whisper-rs-nx는 반대 접근을 취합니다: 프로젝트 컨텍스트를 충분히 풍부하게 만들어 AI 지원 기여가 기본적으로 높은 품질이 되도록 구조적으로 AI 기여를 수용합니다. 이 포크는 업스트림이 AI 작업을 거부하기 때문에 존재합니다.

## 5가지 전제

1. **최소 환경**: AI 코딩 도구 + Git 통합이 최소 요건
2. **양방향 AI**: 기여자와 메인테이너 모두 AI 사용
3. **영어 공용어**: Git의 공개 공유는 영어; 비공개 작업 로그는 모국어
4. **혼합 기술 수준**: 초보자부터 전문가까지; AI가 가이던스를 조정
5. **커뮤니케이션 흐름**: 사람 → AI → Git (영어) → AI → 사람

## 품질 전략

구조가 품질을 보장하며, 게이트키핑이 아닙니다.

| 레벨 | 이름 | 메커니즘 |
|------|------|---------|
| L1 | 컨텍스트 | `.agents/` 디렉토리 — AI가 코드 생성 전 프로젝트를 이해 |
| L2 | 자동화 | CI 게이트 (cargo build, test, clippy, fmt) |
| L3 | 인간 판단 | 아키텍처와 방향에 대한 메인테이너 리뷰 |

## AI 어트리뷰션

- AI 사용을 환영하며 투명성을 감사히 여깁니다
- AI 생성 코드에 대한 책임은 기여자에게 있습니다
- Git 트레일러: `Assisted-by: {도구 이름}` (권장, 필수 아님)
